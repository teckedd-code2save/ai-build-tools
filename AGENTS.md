## 4. Agent Conventions

- **Two `skills/` directories must stay in sync** — `cli/skills/` is a copy of `skills/` for npm packaging. Run `scripts/sync-skills.sh` after adding or updating any skill.
- **Update `ALL_SKILLS` and `SKILL_DESCRIPTIONS`** in `cli/src/constants.ts` whenever a new skill is added.
- **CLI commands are registered** in `cli/src/index.ts` via `register*Command` functions in `cli/src/commands/`.
- **Skill installation** happens through `cli/src/setup/skill-writer.ts` with `fs.cp(source, dest, { recursive: true })`.
- **MCP config** is generated by `cli/src/setup/mcp-writer.ts`.
- **Agent detection** is handled by `cli/src/setup/agents.ts`.
- **Do NOT modify published npm versions** — publish happens automatically on push to `main` via `forge-publish.yml`.


## 6. Build & Deploy

- **Package**: `npm run build` produces `cli/dist/` — published to npm as `@teckedd-code2save/forge`
- **CI**: Every push to `main` triggers `forge-publish.yml` (npm publish + GitHub Pages). Also runs `check-skills-sync.yml` to ensure `skills/` and `cli/skills/` are in sync.
- **Pages**: The repo's GitHub Pages site (live at `https://teckedd-code2save.github.io/ai-build-tools/`) serves the landing page from `index.html`.

## 8. Code Conventions

- **ESM only** — `"type": "module"` throughout. No `require()`, no `__dirname`.
- **Commands use Commander.js** — chain `.command()`, `.description()`, `.argument()`, `.action()`.
- **No direct `console.log` in commands** — use the `logger` utility from `cli/src/utils/logger.ts`.
- **Skill SKILL.md format** — YAML frontmatter with `name`, `description`, `version`, `author`, `license`, `metadata.hermes` tags.
- **Pinned skills cannot be deleted** — use `hermes curator unpin <name>` first for removal.
